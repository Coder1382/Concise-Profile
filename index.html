PET-projects

In the Browser Game a triangle subject travels along the screen among square obstacles that move towards it. The control is performed via arrow keys Left/Right/Up/Down; to travel faster, tap a respective arrow key. In the game, there are three levels differing in the rate of appearance of obstacles. In the level 3, obstacles rotate. You can start playing by following the link file. The project is written in JavaScript, HTML. CSS.
REST API application (JDBC, PostgreSQL) implements One-to-Many and Many-toMany relations, as well as processes requests as follows:
  • show all table records / one table record (GET request with/without parameter)
  • add a table record (POST request)
  • update a table record (PUT request)
  • delete a table record (DELETE request)
There are three table at hand: Fruit, Sellers. Suppliers. A record in the Fruit table includes of two mandatory fields: Name and Price. Each record has a list of sellers. User can update the price, as well as add/remove sellers to/from the list.
A record in the Sellers table includes one mandatory field Name, and one optional field Supply. Each record has a list of fruit traded by a seller. User can change a supplier for a certain seller, as well as add/remove fruit to/from the list.
A record in the Suppliers table includes one mandatory field Name. Each record has a list of sellers served by such a supplier. User can add/remove sellers to/from the list.
Upon adding, editing, removal records in tables, lists associated with respective records get automatically updated/deleted.
Examples of making use of this application are provided in the README.md file. The project is written in Java.

The Linux Socket project is a console application for establishing a TCP connection and data exchange between a server and clients. Message sent by a client via the console are dispalyed in the server's console. The server saves such an acquired message in a log-file, and echoes it to the client's console.
The server's console requires one input parameter: the port number. The client's console requires an input string comprising three parameter: the client name, port number, and the connection time in seconds. An example of making use of the project is provided in the README.md. The project is written in C++.
